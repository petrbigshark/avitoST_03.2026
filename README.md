# ST_v2 - 03.2026: Автосборка готового файла Avito

Этот проект собирает финальный файл для автозагрузки Avito из нескольких Excel-источников, подтягивает данные по артикулам, формирует описания, загружает фото в IMGBB и сохраняет результаты в отдельную папку запуска.

## Что делает скрипт

Скрипт `build_ready_file.py` выполняет полный пайплайн за один запуск:

1. Читает источники:
- `Avitonow.xlsx`
- `Base.xlsx`
- `Отсутствующие в base.xlsx` (fallback для артикулов, которых нет в `Base.xlsx`)
- `Готовый файл.xlsx` (шаблон для заполнения)

2. Сопоставляет строки по полю:
- `Уникальный идентификатор объявления`

3. Заполняет поля в листе `Женская обувь-Кроссовки и кеды`:
- `Описание объявления` (по шаблону)
- `Размер` (строго из `Avitonow -> Размер: RU`)
- `Вид одежды` (из `Base: GoodsType` или fallback)
- `Цвет` (из `Base: Color` или fallback)
- `Вид одежды, обуви, аксессуаров` (из `Base: ApparelType` или fallback)
- `Категория` (подменяется из fallback, если в `Отсутствующие в base.xlsx` она задана)
- `Ссылки на фото` (из IMGBB)

4. Загружает фото в IMGBB:
- Берёт файлы только изображений
- Видео (`.mov` и другие видеоформаты) не загружает
- HEIC/HEIF по умолчанию конвертируются в JPEG (`sips formatOptions=best`) перед загрузкой, чтобы ускорить прогон и снизить 400-ошибки
- Если IMGBB отклоняет файл с `400`, скрипт автоматически делает fallback: конвертирует только этот файл в JPEG (`sips formatOptions=best`) и повторяет загрузку
- Для нескольких фото ставит разделитель `|`

5. Формирует 3 выходных файла:
- `Готовый файл <дата> vN.xlsx`
- `IMGBB ссылки <дата> vN.xlsx`
- `Лог <дата> vN.txt`

6. Кладёт их в отдельную папку запуска:
- `output Готовые файлы/Готовые файлы <дата> vN/`

## Что нужно в папке проекта

В директории `ST_v2 - 03.2026` должны быть:

- `build_ready_file.py`
- `Avitonow.xlsx`
- `Base.xlsx`
- `Отсутствующие в base.xlsx` (если есть fallback-артикулы)
- `Готовый файл.xlsx`
- папка `Avito_ST_2` с подпапками по артикулам и фотографиями

## Папка с фото (Avito_ST_2)

- В Git хранится только пустая папка `Avito_ST_2` (через `.gitkeep`), сами фото/видео в репозиторий не загружаются.
- Фото для `Avito_ST_2` нужно скачать отсюда: [Яндекс.Диск](https://disk.yandex.ru/d/1OcBT4ten7keFg)
- Имена подпапок в `Avito_ST_2` должны быть **только артикулами**, без дополнительных слов/пометок.

## Быстрый запуск

### 1) Установить ключ IMGBB в текущий терминал

```bash
export IMGBB_API_KEY='ВАШ_КЛЮЧ_IMGBB'
```

### 2) Запустить полный процесс

```bash
python3 -B "/Users/petr/Desktop/Projects/Avito_ST/ST_v2 - 03.2026/build_ready_file.py"
```

### 3) Где смотреть результат

В папке:

`/Users/petr/Desktop/Projects/Avito_ST/ST_v2 - 03.2026/output Готовые файлы/Готовые файлы <дата> vN/`

## Тестовый запуск без реальной загрузки фото

Если нужно проверить логику без обращения к IMGBB:

```bash
python3 -B "/Users/petr/Desktop/Projects/Avito_ST/ST_v2 - 03.2026/build_ready_file.py" --skip-upload
```

В этом режиме в `Ссылки на фото` будут mock-ссылки (`mock://...`).

## HEIC-конвертация (опционально)

По умолчанию HEIC/HEIF конвертируются в JPEG (`formatOptions=best`) перед загрузкой.

## New Link режим

Если нужно принудительно получить новые ссылки в выгрузке:

```bash
python3 -B "/Users/petr/Desktop/Projects/Avito_ST/ST_v2 - 03.2026/build_ready_file.py" --newlink
```

Это создаст прогон со следующей версией (vN+1).

Если нужно перезаписать конкретную версию:

```bash
python3 -B "/Users/petr/Desktop/Projects/Avito_ST/ST_v2 - 03.2026/build_ready_file.py" --newlink v1
```

Поддерживаются форматы `v1` и `1`.

## Прогресс в терминале

Скрипт печатает:

- старт: сколько папок с фото и сколько фото в обработке;
- прогресс каждые 100 фото (`📊 Progress: ...`);
- ошибки загрузки фото по мере появления (`❌ Upload error ...`);
- итоговые пути к 3 файлам;
- общее время выполнения в секундах.

## Fail-fast по ошибкам фото

По умолчанию скрипт остановится, если будет `10` ошибок загрузки фото подряд.

Параметр запуска:

```bash
python3 -B "/Users/petr/Desktop/Projects/Avito_ST/ST_v2 - 03.2026/build_ready_file.py" --max-consecutive-photo-errors 10
```

Отключить остановку по подряд идущим ошибкам:

```bash
python3 -B "/Users/petr/Desktop/Projects/Avito_ST/ST_v2 - 03.2026/build_ready_file.py" --max-consecutive-photo-errors 0
```

## Что если товар продан

Если по артикулу было несколько размеров и один размер продали:

- удалите соответствующую строку из `Avitonow.xlsx`;
- запустите скрипт снова;
- в новом прогоне размеры в описании пересчитаются по актуальным оставшимся строкам.

## Откуда брались 3 файла вне папки

Это наследие старого формата (когда файлы сохранялись в корень `output Готовые файлы`).

Теперь скрипт при запуске автоматически переносит такие старые файлы в нужную папку версии. Если в папке версии уже есть файл с таким же именем, старый файл переносится с суффиксом `(flat-root)`.

## Логи и диагностика

Файл `Лог <дата> vN.txt` содержит:

- входные файлы;
- выходные пути;
- список предупреждений по Base;
- runtime-сообщения;
- ошибки загрузки фото;
- итоговое время выполнения.

Если есть `!НЕТ!` в итоговом файле:

- проверьте артикул в `Avitonow.xlsx`;
- проверьте соответствие артикула в `Base.xlsx` или `Отсутствующие в base.xlsx`;
- проверьте наличие папки артикула в `Avito_ST_2`;
- проверьте доступ в интернет и корректность `IMGBB_API_KEY`.

## Примечание по предупреждению openpyxl

Сообщение вида:

`UserWarning: Data Validation extension is not supported and will be removed`

это предупреждение библиотеки `openpyxl`, не критичная ошибка. Обычно на результат заполнения не влияет.
